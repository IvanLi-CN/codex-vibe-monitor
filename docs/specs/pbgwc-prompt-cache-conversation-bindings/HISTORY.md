# Prompt Cache Conversation Bindings - History

## Key Decisions

- 2026-05-25: Created a dedicated topic spec because per-conversation routing bindings introduce a new stable runtime contract distinct from invocation observability.
- 2026-05-25: Chose hard-constraint routing semantics so bound conversations never silently fall back to unrelated accounts or groups.
- 2026-05-25: Added Storybook coverage and visual evidence for the drawer binding panel to keep the UI contract reviewable.
- 2026-05-27: Clarified that manual Prompt Cache account and group bindings are the only supported way to move a conversation out of a sticky source whose policy forbids cut-out; group targets still honor target cut-in policy.
- 2026-05-28: Locked retained-history drawer paging to 50 records per page and moved `InvocationTable` to active-breakpoint virtualization after a 15,000-record drawer proved that full-page hydration and dual hidden layouts could block route-binding controls.
- 2026-06-29: Expanded the binding row contract to support timeout-only `binding_kind='none'` rows. Conversation timeout overrides now persist independently from the manual binding target, are applied after target account selection, and do not trip encrypted-session owner guard logic.
- 2026-07-01: Aligned the conversation timeout editor with the effective-rule interaction model. Timeout rows now render as collapsed summaries until the conversation layer overrides a field, while timeout-only `binding_kind='none'` rows still preserve their explicit per-field override state.
- 2026-07-02: Expanded conversation bindings into runtime policy overrides. The Settings tab can now override upstream switching (“切出”), FAST mode, image tool, available models, and one existing forward-proxy binding node for the current prompt cache key.
- 2026-07-13: FAST mode and image tool Select menus now show only concrete rewrite modes. Successful choices retain the field-local editor instead of collapsing it, while the existing explicit clear action continues to restore inheritance.
- 2026-07-15: Re-aligned the conversation detail drawer with the routing spec after the wide-shell/shared-form regression resurfaced on the live dashboard. The Settings tab now reuses the same effective-routing rule card skeleton as account routing, keeps the route-binding block separate, hides account-only rows on the conversation surface, and keeps the desktop overlay frame full-width so the shared wide detail shell no longer shrinks or expands with tab content.
- 2026-07-17: Followed up on the post-release regression where the shared wide drawer still rendered nearly full-viewport width. The root cause was the shared shell's base `w-full` utility overriding the wide modifier in the final CSS cascade; the modifier now wins by selector specificity, and Storybook now asserts the actual desktop pixel width instead of only checking for the class name.
- 2026-07-17: Added Dashboard-scoped conversation multi-select and bulk actions. Operators can toggle `选择模式` or use temporary `Cmd`/`Ctrl` card selection, then bulk-bind selected conversations to a group or upstream account, clear and reset conversation affinity by removing the manual binding plus sticky/owner locks, or batch-write FAST rewrite mode without changing existing binding kind.
- 2026-07-20: Split Dashboard bulk clear binding from clear/reset affinity. The Dashboard destructive clear action now sends bulk `bind` with `bindingKind='none'`, clearing only the manual binding target while preserving sticky route and encrypted owner lock rows; `clearAndResetAffinity` remains a separate API action for full affinity reset callers.
- 2026-07-20: Fixed the follow-up color regression on the Dashboard bulk clear confirmation dialog. Theme-derived chrome and destructive callout tokens are now scoped to the active themed ancestor instead of only `:root`, and the Storybook dark-theme surface keeps `html/body` theme attributes aligned so the dialog cannot mix dark body content with light header/footer chrome.
- 2026-07-03: Expanded conversation proxy overrides from one node to a hard multi-node list. Runtime keeps the current node sticky per prompt cache key, switches within the explicit list after consecutive network failures, and never falls back to account/group/automatic proxy routing while the conversation list is set.
- 2026-07-07: Locked automatic sticky escape to non-explicit routing only. After the latest two terminal pool `/v1/responses` attempts on the same account both fail as `upstream_stream_error`, automatic routing stops reusing that account across sticky keys, `group` bindings may reselect another account inside the group, and explicit upstream-account bindings remain operator overrides.
- 2026-07-18: Added a sibling `操作记录` tab to Prompt Cache conversation detail and locked the v1 presentation to one event stream plus lightweight badges/filters instead of nested subtabs.
- 2026-07-18: Normalized per-conversation operation history into three categories: `routing`, `forwardProxy`, and `requestRewrite`. Policy PATCH records can carry multiple categories in one `infoTypes[]` payload so mixed-field writes stay collapsed.
- 2026-07-18: Locked operation origins to `detailDrawer`, `dashboardBulk`, and `systemAuto`, while explicitly excluding historical backfill and actor-identity audit from v1.
- 2026-07-18: Deduplicated Storybook-local React Vite plugins so the `DrawerOperations` story can build and serve stable mock visual evidence for the new operations tab.
- Upstream account bindings are operator-forced account assignments: they override sticky transfer policy only, not account health, quota, guard, concurrency, route-key, or forward-proxy readiness.
- “切出” is a conversation-level permission to switch away from the original/sticky upstream account, preserving user-facing terminology from account settings without introducing a conversation-level cut-in permission.
