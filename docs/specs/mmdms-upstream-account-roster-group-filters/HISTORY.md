# 上游账号列表分组筛选 演进历史（#mmdms）

> 这里记录会影响 Agent 理解“为什么一步步变成现在这样”的关键演进；单次任务流水账不放这里，规范正文仍以 `./SPEC.md` 为准。

## Decision Trace

- 账号列表分组筛选从单选/自由文本改为精确多选，以支持跨分组运营查看。
- `未分组` 从 UI 伪项改为真实默认分组名，避免空分组在不同界面和运行时语义不一致。
- 分组候选口径固定为全量有账号分组，避免其它筛选条件让可选分组消失。

## Key Reasons / Replacements

- 自由文本搜索会让“候选过滤”和“列表过滤”语义混杂；精确枚举能保证筛选结果可解释。
- 空的仅备注分组仍属于分组管理主题，不应进入账号列表筛选候选。
